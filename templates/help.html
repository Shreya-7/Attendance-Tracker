This describes the general flow of using this webapp. For more details, navigate to the respective tabs.

  1. Before we start tracking attendance, a course has to be added, assigned to a batch and students added to it. This is a one-time setup for every course.
  2. To track attendance for a course, an attendance report from MS Teams or responses from a Google form has to be attached. That's it.
  3. You can also upload multiple reports at a time and mix-match the two formats available.
  4. To view the tracked attendance for a course, reports can be downloaded in multiple formats. These reports can either be regular or defaulter, and are automatically updated till the latest date for which attendance has been tracked.
  5. To no longer track attendance for a course, it can be deleted. This is an irreversible process.
  6. Meticulous care has been taken to catch and display any errors regarding user action with detailed information. If any such error is not caught, please report it back to us.

A course is uniquely identified by its Course ID and Batch. This means you can track attendance for the same course across multiple batches.

  • Navigate to the Add Course section of the homepage. It consists of a single form.
  • Fill in the Course ID. For example, CS402
  • Fill in the Course Name. For example, Data Structures and Algorithms
  • Fill in the Batch Year. For example, 2017
  • Upload the list of students who have to be tracked under this course. The file (CSV/XLSX) has to be uploaded in the following format:
    NameRoll Number
    Shreya 10
    Ranesha 11
    Ritwik 12
    Riya 13
    Any and all tracking done for the student is via the Roll Number.
  • Very Important Note: The Full Name field in the MS Teams report and the Your Roll Number field in the Google Form report should match the Roll Number field of this student file, not the Name field. For more info on this, please check the Upload A Report tab.
  • Click on the Create Course button.
  • Below this button, a status update will be displayed. If it's successful, then the course has been added. Otherwise, a message stating the mistake will be shown and you can re-do the process.

Attendance can be uploaded here and their parsing logic can be controlled by a number of features.

  • Navigate to the Upload Attendance section of the homepage.
  • Select the Course from the dropdown. All identifying details like Course ID, Batch will be visible for you to clearly identify the course.
  • Select the kind of upload to be done:
    • Batch - this mode allows you to upload multiple reports at a time. You can upload both MS Teams and Google Form reports at the same time. No threshold or flag features are applicable here. The format for both reports have been described below.
    • Single - this mode allows you to upload only a single file at a time. You can choose between 2 formats, both described below. You can get a lot more control over the tracking in this mode as you can use both threshold and flag features depending on the format you choose.
  • Select the kind of report to be uploaded (if opting for Single Upload):
    • MSTeams - this is the downloadable report available on the platform. The parsing logic allows for multiple entries for the same student to accomodate disconnections due to technical issues.
    • Google Form - responses from a Google form extracted into a CSV or XLSX file are also accepted.
      • This mode however, disables the Threshold feature.
      • Columns can be interchanged. Any extra columns are ignored.
      • Multiple entries for the same student are ignored.
  • Upload the file(s) conforming to the format:
    • MSTeams - The format (CSV/XLSX) should be as follows:
      Full Name User Action Timestamp
      10 Joined 11/2/2021, 9:35:35 AM
      11 Joined 11/2/2021, 9:40:35 AM
      10 Left 11/2/2021, 9:42:35 AM
      12 Joined 11/2/2021, 9:50:35 AM
    • Google Form - The format (CSV/XLSX) should be as follows:
      Timestamp Your Roll Number
      2021/02/25 12:13:03 PM GMT+5:30 10
      2021/02/25 12:15:03 PM GMT+5:30 11
  • Fill in the Meeting End Time (applicable only for single upload MS Teams reports).
    • This will be used to track the duration of the class.
    • The Meeting Start Time is assumed to be the smallest timestamp in the file.
    • Format: 11/2/2020, 9:35:35 AM
  • Fill in the Threshold in minutes (applicable only for single upload MS Teams reports):
    • This is used to mark students present only if they have been logged into the class for atleast this amount of time.
    • Helps to weed out students who have a log in the upload file only by joining and leaving quickly.
  • Fill in the Flagged Roll Numbers (applicable only for single uploads):
    • This is a comma-separated list of roll numbers for whom attendance will not be tracked even if they meet the threshold criteria.
    • Helps to forcibly mark those students as absent who the teacher feels was just logged into the class but not listening/active/responsive.
  • Click on the Upload button.
  • Below this button, a status update will be displayed. If it's successful, then the course has been added. Otherwise, a message stating the mistake will be shown and you can re-do the process.

The attendance reports uploaded for a course are parsed and collated into one single file for the user to consume. There are multiple options you can choose from when it comes to downloading these reports.

  • Navigate to the Download Attendance section on the homepage.
  • Select the course from the dropdown. All identifying details like Course ID, Batch will be visible for you to clearly identify the course.
  • Select the kind of report needed: Complete shows the complete data for all the days and Defaulter shows only the date-wise data for students having less than 50% total attendance till the latest date of tracking.
  • Select the format of the report needed: CSV or XLSX.
  • Click on the Download button.
  • You will be prompted to save the file renamed as CourseID_Batch_asOf_LastTrackingDate.Format It will also include the Defaulter postfix if that is selected.
  • The downloaded file for a course with 3 tracked dates will look like the following:
    Roll Number Date1 Date2 Date3
    10 TRUE TRUE TRUE
    11 TRUE FALSE TRUE
    12 TRUE FALSE FALSE
    13 TRUE TRUE FALSE
    • These dates are sorted in ascending order, no matter the order in which they were uploaded.
    • TRUE indicates that the person was present on that day.
    • The Defaulter list looks the same except it only contains details of the students having less than 50% total attendance till the latest date of tracking.

If you do not wish to track attendance for a course any more, it can be deleted. It is an irreversible process and will delete all data and files associated with the course. As a fallback, you can download the latest report before proceeding with the steps below.

  • Navigate to the Delete Course section on the homepage.
  • Select the course you wish to delete from the dropdown. All identifying details like Course ID & Batch will be visible for you to clearly identify the course.
  • Click on the Delete Course button.
  • Below the button, you will get a confirmatory message.
